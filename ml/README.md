# Bezum Hack 2026: ML Meeting Insights
Система автоматического протоколирования и интеллектуального анализа видеовстреч.

## Цель проекта
Создание платформы для мониторинга корпоративных коммуникаций и автоматического формирования отчетности на базе искусственного интеллекта. Решение преобразует сырые видеозаписи в структурированные знания, минимизируя рутинную работу сотрудников по итогам созвонов.

## О проекте
Наша команда представляет сервис для автоматической сегментации спикеров и расшифровки речи. Решение автоматизирует формирование протоколов встреч, а мощные модели машинного обучения позволяют проводить глубокий семантический анализ текста. Благодаря интеграции с LLM, система выделяет ключевые договоренности и задачи, позволяя оперативно контролировать исполнение поручений. Краткая форма отчета позволяет быстро ознакомиться с основными сведениями о встрече за любой заданный период.

Стек решения: Python, FastAPI, Pyannote 3.1, GigaAM-v3, Gemma 3 (Ollama), RabbitMQ, Docker, MinIO (S3).

Уникальность решения заключается в использовании каскадной архитектуры моделей разного типа (Diarization + STT + LLM), что в совокупности создает эффективный инструмент для бизнес-мониторинга.

## Функционал

### Инфраструктура и Деплой
* Развертывание через Docker-compose.
* Очереди задач на базе RabbitMQ для асинхронной обработки.
* Хранение артефактов (аудио, протоколы, саммари) в S3-совместимом хранилище.

### Мониторинг и Анализ (ML)
* Диаризация: Идентификация уникальных голосов и разметка таймкодов.
* Транскрипция: Высокоточное распознавание русской речи с использованием GigaAM-v3.
* Ассистент: Генерация текста на основе запроса к LLM.
* Саммаризация: Составление краткого отчета на основе стенограммы.
* Экстракция задач: Автоматическое выделение Action Items (исполнитель, задача, срок).

### Отчетность
* Формирование протоколов в формате Markdown.
* Сбор метрик производительности (время обработки, потребление памяти).
* Возможность скачивания готовых отчетов через API.

## Методология обработки
1. Извлечение аудиодорожки из видеофайла.
2. Диаризация (Pyannote): Построение карты присутствия спикеров.
3. Транскрипция (GigaAM): Посегментный перевод речи в текст.
4. Саммаризация (Gemma): Формирование кратких выводов и списка задач.
5. Экспорт: Загрузка финальных документов в объектное хранилище.

## Get Started

1. Настройка окружения: Укажите необходимые ключи и адреса сервисов в файле .env.
2. Запуск контейнеров: 
   docker-compose up --build -d
3. Загрузка языковой модели:
   docker exec -it ollama_service ollama run gemma3:1b
